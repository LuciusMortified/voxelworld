# Сводка по дизайну воксельного движка на Vulkan

## Что было спроектировано

Была создана полная архитектура воксельного движка на Vulkan с модульной структурой, включающая:

### 🏗️ Архитектурные компоненты

#### 1. **Основа движка** (`engine.h/cpp`)
- Главный координатор всех подсистем
- Игровой цикл и управление временем
- Инициализация и очистка ресурсов

#### 2. **Система окон** (`window.h`)
- Интеграция с GLFW для создания окон
- Управление Vulkan surface
- Обработка событий ввода

#### 3. **Vulkan контекст** (`vulkan_context.h`)
- Инициализация Vulkan instance
- Выбор и настройка GPU устройства
- Управление очередями команд
- Создание command pool

#### 4. **Рендер система** (`renderer.h`)
- Управление swapchain
- Создание render pass и graphics pipeline
- Синхронизация кадров
- Обработка изменения размера окна

### 🎮 Игровые объекты

#### 5. **Мир** (`world.h`)
- Контейнер для воксельных моделей
- Управление позициями объектов
- API для добавления/удаления моделей

#### 6. **Модели** (`model.h`)
- 3D массив вокселей
- Операции установки/получения вокселей
- Эффективное хранение данных

#### 7. **Воксели** (`voxel.h`)
- Простая структура с упакованным цветом
- Базовый строительный блок

### 🎥 Камера и управление (`camera.h`)
- View и projection матрицы
- Управление позицией и ориентацией
- FPS-стиль движения

### 🔧 Вспомогательные системы

#### 8. **Буферы** (`buffer.h`)
- Базовый класс для Vulkan буферов
- Специализированные классы: vertex_buffer, index_buffer, uniform_buffer
- Управление GPU памятью

#### 9. **Мешы** (`mesh.h`)
- Представление геометрии для GPU
- Генератор мешей из воксельных моделей
- Оптимизация через face culling

#### 10. **Шейдеры** (`shader.h`)
- Загрузка SPIR-V шейдеров
- Управление shader modules
- Создание pipeline stages

### 🎨 Шейдерная программа

#### Vertex Shader (`voxel.vert`)
```glsl
- Входные данные: Position, Normal, Color (упакованный)
- Uniform: Model/View/Projection матрицы + освещение
- Трансформация вершин в clip space
- Распаковка цветов
```

#### Fragment Shader (`voxel.frag`)
```glsl
- Модель освещения Фонга (ambient + diffuse + specular)
- Эффект тумана на основе расстояния
- Финальная композиция цвета
```

## 🚀 Рендер пайплайн

### Этапы рендеринга:
1. **Подготовка данных**: Voxel Model → Mesh Generator → GPU Buffers
2. **Begin Frame**: Получение изображения из swapchain
3. **Update Uniforms**: Обновление матриц и освещения
4. **Bind Pipeline**: Привязка graphics pipeline
5. **Render World**: Отрисовка всех моделей в мире
6. **End Frame**: Презентация кадра

### Vulkan Pipeline:
```
Vertex Input → Vertex Shader → Primitive Assembly → 
Rasterization → Fragment Shader → Color Blending → Framebuffer
```

## 🎯 Ключевые особенности

### ✅ Оптимизации
- **Face Culling**: Скрытые грани не генерируются
- **Staging Buffers**: Эффективное копирование данных
- **Command Buffer Reuse**: Переиспользование команд
- **Packed Colors**: Компактное хранение цветов (32-bit)

### ✅ Современная архитектура
- **RAII**: Автоматическое управление ресурсами
- **Move Semantics**: Эффективное перемещение объектов
- **Template Specialization**: Типобезопасные буферы
- **Модульность**: Четкое разделение ответственности

### ✅ Расширяемость
- Готова для добавления texture atlas
- Поддержка instanced rendering
- Возможность добавления compute shaders
- Архитектура позволяет легко добавлять новые эффекты

## 📊 Структура данных

### Vertex (28 bytes):
```cpp
struct vertex {
    vec3f position;  // 12 bytes
    vec3f normal;    // 12 bytes  
    uint32 color;    // 4 bytes (RGBA packed)
};
```

### Uniform Buffer (240 bytes):
```cpp
struct uniform_buffer_object {
    mat4 model;      // 64 bytes
    mat4 view;       // 64 bytes
    mat4 projection; // 64 bytes
    vec3f view_pos;  // 16 bytes (aligned)
    vec3f light_pos; // 16 bytes (aligned)
    vec3f light_color; // 16 bytes (aligned)
};
```
